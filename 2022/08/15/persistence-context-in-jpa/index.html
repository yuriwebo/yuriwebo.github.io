Persistence Context in JPA

yuriwebo
2022-08-15
Coding - Knowledge
JPA, Persistence Context

Persistence Context: Describes the relationship between all the Entity instances in our program and their representations in the underlying database.

Udacity's Glossary

Entities are managed by javax.persistence.EntityManager instance using Persistence Context.

Each EntityManager instance is associated with a Persistence Context.

Within the Persistence Context, the entity instances and their lifecycle are managed.

Persistence Context defines a scope under which particular entity instances are created, persisted, and removed.

A Persistence Context is like a cache which contains a set of persistent entities, so once the transaction is finished, all persistent objects are detached from the EntityManagerâ€˜s Persistence Context and are no longer managed.

Summarily, a Persistence Context handles a set of entities which hold data to be persisted in some persistence store (e.g. a database). In particular, the context is aware of the different states an entity can have (e.g. managed, detached) in relation to both the context and the underlying persistence store.
